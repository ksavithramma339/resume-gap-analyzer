Today I am excited to show you the Resume Gap Analyzer, a tool designed to solve a major problem for job seekers: how to safely and effectively optimize resumes for applicant tracking systems, verify format compliance, and locate high-quality courses to close skills gaps, all while protecting private personal data.

Resume Gap Analyzer is an intelligent, secure multi-agent workflow built with the Gemini Enterprise Agent Platform and the ADK 2.0 SDK. It empowers candidates to improve their career opportunities securely.

As you can see on our cover banner, the system analyzes resume text against target job descriptions, checks compliance via MCP, and retrieves tailored course recommendations to fill skill gaps while enforcing strong security checkpoints.

(show assets/architecture_diagram.png)
Let's walk through the architecture diagram to see how it works under the hood.
First, the input resume and job description go to the Security Checkpoint node. This node scrubs candidate PII, checks for prompt injection attacks, and performs domain-specific validation.
If the check passes, we proceed to the Orchestrator agent, which coordinates the work.
The Orchestrator delegates tasks to specialized sub-agents: the Resume Analyzer extracts resume sections and skills, and the Gap Analyzer compares them to the job description to find missing skills.
The Gap Analyzer uses our custom python-based MCP Server's tool verify_resume_format to verify formatting constraints.
Next, we reach the Ask Learning Paths Human-in-the-Loop node. This pauses execution and prompts the user for consent.
If the user consents, the Gap Enhancer agent is triggered. It queries the MCP Server tools search_courses and fetch_job_trends to find highly relevant courses and market stats.
Finally, the Final Report node compiles a comprehensive, secure markdown report.

The Security Checkpoint safeguards the user and the system by scrubbing emails, SSNs, and phone numbers, detecting adversarial prompts, and generating structured JSON logs.
Our custom FastMCP server runs on stdio transport and exposes three tools: verify_resume_format to evaluate ATS compatibility, search_courses to fetch target courses, and fetch_job_trends to pull salary and demand stats.
